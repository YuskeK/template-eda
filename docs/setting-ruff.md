# Ruff設定ガイド

## 基本設定

### line-length
1行あたりの最大文字数を指定する。

- `88`: Blackのデフォルト値
- `120`: より長い行を許容する設定

## [tool.ruff.lint]

### select - 有効化するルール

#### `"E"` - pycodestyle errors
PEP 8スタイルガイドの構文エラーをチェックする。
- `E501`: 行が長すぎる
- `E302`: 関数間に2行空行がない

#### `"W"` - pycodestyle warnings
エラーほど重大ではないが修正すべき問題を検出する。
- `W291`: 行末に空白がある
- `W605`: 無効なエスケープシーケンス

#### `"F"` - pyflakes
コードの論理的な問題を検出する。
- `F401`: 未使用のインポート
- `F841`: 未使用の変数

#### `"I"` - isort
インポート文の並び順をチェック・整理する。  
標準ライブラリ → サードパーティ → 自作モジュールの順に並べる。

#### `"B"` - flake8-bugbear
バグを引き起こしやすいパターンを検出する。
- `B006`: 可変デフォルト引数
- `B008`: 関数呼び出しをデフォルト引数に使用

#### `"C4"` - flake8-comprehensions
内包表記を最適化する。
- `list()` より `[]` を推奨
- 不要な `list()`/`dict()` 呼び出しを検出

#### `"UP"` - pyupgrade
古い構文を新しいPythonの機能に更新する。
- `%` フォーマットを `f-string` に変換
- `typing.List` を `list` に変換

---

### ignore - 無視するルール

#### `"E501"` - 行が長すぎる
`line-length` で制御するため重複を避ける。フォーマッターに任せる。

#### `"B008"` - デフォルト引数での関数呼び出し
FastAPIの `Depends()` などで必要なため無視する。

#### `"C901"` - 関数が複雑すぎる
Cyclomatic Complexityが高い関数を検出するが、実用上は許容することが多い。

#### `"W191"` - インデントにタブが含まれる
タブとスペースの混在を許容する（プロジェクト方針による）。

#### `"B904"` - raise ... from ... の形式
元の例外を追跡するため `from` を使うべきだが、無視する。

---

### unfixable
自動修正を禁止するエラーコード。

```python
["F401", "F403"]  # 未使用インポート、import *
```

削除されると困る場合があるため、手動で判断する。

---

## [tool.ruff.lint.per-file-ignores]

特定のファイルで特定のルールを無視する設定。

```python
"__init__.py" = ["F401"]
```

`__init__.py` では再エクスポートのために未使用に見えるインポートが必要となる。

---

## [tool.ruff.lint.isort]

### known-first-party
自作パッケージ名を指定する。

```python
["src"]  # src配下を自作モジュールとして認識
```

### known-third-party
サードパーティパッケージを明示する。

```python
["fastapi", "pydantic"]
```

### section-order
インポートセクションの順序を定義する。

---

## [tool.ruff.lint.pyupgrade]

### keep-runtime-typing
実行時型情報を保持する。

- `True`: `typing.List` → `list` への変換を抑制（Python 3.8との互換性）
- `False`: 積極的に新しい型ヒントに変換
